# Flexcil Viewer

Браузерный просмотрщик заметок из приложения [Flexcil](https://www.flexcil.com/).

# ВНИМАНИЕ! Это прототип, сделанный с помощью ИИ, код ещё не отсмотрен, не в коем случае не открывать файлы/папки без резервной копии. Также советую ставить права read-only на папки/файлы, чтобы программа их не затёрла в случае чего

## Возможности

- 📄 Просмотр документов `.flx` (заметки Flexcil)
- 📁 Загрузка целой папки бэкапа Flexcil (в том числе папки Flexcil Sync)
- 🖊️ Отображение рукописных аннотаций поверх PDF
- 🖼️ Поддержка вставленных изображений
- 🎵 Воспроизведение аудиозаписей (`.fab`) с синхронизацией рисунков
- 🔍 Зум и навигация по страницам
- 🎨 Переключение слоёв (PDF/рисунки/изображения)

## Запуск

### Способ 1: Простой HTTP сервер

```bash
cd viewer
python3 -m http.server 8080
```

Затем откройте http://localhost:8080

### Способ 2: Node.js

```bash
cd viewer
npx serve
```

### Способ 3: VS Code Live Server

Установите расширение "Live Server" и откройте `viewer/index.html`

## Использование

1. Нажмите "Открыть папку бэкапа" для загрузки папки `Flexcil Backup ...`
2. Или нажмите "Открыть .flx файл" для загрузки отдельной заметки
3. Навигация: стрелки влево/вправо или кнопки
4. Зум: Ctrl+/- или кнопки увеличения
5. Пробел: воспроизведение/пауза аудио

## Формат файлов Flexcil

### .flx (заметка)
ZIP-архив со структурой:
- `info` - JSON метаданные документа
- `pages.index` - JSON список страниц
- `attachment/PDF/` - PDF фоны страниц
- `attachment/image/` - Вставленные изображения (PNG)
- `objects/<pageKey>.drawings` - Рисунки страницы (JSON с Base64 точками)
- `objects/<pageKey>.images` - Метаданные изображений
- `thumbnail*` - Превью документа

### .fab (аудиозапись)
ZIP-архив со структурой:
- `<id>.flxa` - Аудио файл (MP4/AAC)
- `<id>.rinfo` - JSON метаданные записи
- `<id>.rsync` - JSON синхронизация с рисунками

### Формат точек рисунка
Base64-закодированные бинарные данные:
- Байт 0: количество точек
- Байты 1-3: padding
- Каждые 12 байт: точка (3 × float32 little-endian: x, y, flags)

Координаты `x` и `y` - смещение относительно `start.x` и `start.y` в JSON.
